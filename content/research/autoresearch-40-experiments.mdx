## The Framework

In March 2026, Andrej Karpathy released [autoresearch](https://github.com/karpathy/autoresearch) - a framework for autonomous AI experimentation. Give an AI agent a training loop and let it iterate overnight.

We adapted it for funding rate strategy optimization.

### How It Works

1. **`prepare.py`** - Fixed. Loads 9.4M funding rate settlements, splits train/val, defines the scoring metric.
2. **`strategy.py`** - The file the AI modifies. Contains all strategy logic: entry/exit rules, position sizing, filtering, risk management.
3. **`program.md`** - Instructions for the AI agent. Domain context about funding rates, fee structures, and anti-patterns.

The AI agent modifies `strategy.py`, runs the backtest (~2 minutes), checks if the score improved, keeps or discards, and repeats. Autonomously.

### The Scoring Metric

A composite of:
- Risk-adjusted return (APY relative to max drawdown)
- Consistency (fraction of profitable days)
- Activity (minimum trades required)
- Fee efficiency (penalizes if fees > 30% of gross)

Both train and validation scores are computed. If train >> val, it's overfitting.

## What We Tested

### Strategies (8 variants)

| # | Strategy | Key Idea |
|---|----------|----------|
| 1 | Spot-vs-perp v3 | Buy spot + short perp, dynamic allocation |
| 2 | Cross-exchange delta | Long low-rate exchange, short high-rate |
| 3 | Interval arb | Exploit settlement frequency differences |
| 4 | Sniper | Trade individual settlements |
| 5 | Event-driven | Only trade extreme rate spikes |
| 6 | Hybrid | Systematic core + event overlay |
| 7 | Delta signal | v3 + cross-exchange signal for timing |
| 8 | Margin mode | Capture negative funding via borrowing |

### Parameter Sweeps

For the winning strategy (spot-vs-perp v3), we swept:
- **Lookback:** 1, 3, 7, 14, 30, 365 days → **1 day optimal**
- **Max positions:** 5, 10, 15, 20, 30 → **20 optimal**
- **Exit decay:** 0.1, 0.3, 0.5, 0.7 → **0.1 optimal** (hold longer)
- **Min rate:** 0.01%, 0.03%, 0.05%, 0.1% → **0.03% optimal**
- **Position cap:** 10%, 15%, 20%, 25% of equity → **20% optimal**

### What Worked

1. **Dynamic allocation** - the single biggest improvement. Fixed $20K → proportional sizing nearly doubled CAGR.
2. **1-day lookback** - funding rate signals are short-lived. Longer lookbacks smooth out the signal too much.
3. **Low exit threshold (0.1×)** - holding positions longer lets costs amortize. Cost ratio drops from 18% to 8%.
4. **Spot availability filter** - removing untradeable symbols improves the backtest's realism without hurting returns.

### What Didn't Work

1. **Delta signal** - using cross-exchange rate divergence as an entry timing signal. Marginal improvement (+2% CAGR) with significant complexity.
2. **Event detection** - extreme rate spikes are too rare to base a strategy on. Capital sits idle between events.
3. **Multi-timeframe** - combining short-term entry with long-term trend confirmation. No improvement over single timeframe.
4. **Exchange bonuses** - weighting Hyperliquid higher (1.3-1.5×) for its hourly settlements. Tiny improvement, not worth the complexity.

## Key Insight

The biggest returns came from **removing complexity**, not adding it:
- Remove fixed position sizes → dynamic allocation
- Remove long lookbacks → 1-day reactivity  
- Remove cross-exchange structure → single-exchange spot-vs-perp

The AI agent spent 20+ experiments adding clever features. The best results came from 3 experiments that simplified.

## Reproducibility

All experiment code is in the `autoresearch/` directory of the Beta-X1 repository. The data (9.4M settlements) is stored in ClickHouse and can be exported for independent verification.

Results are logged in `autoresearch/results.tsv` with commit hashes linking to exact code states.
