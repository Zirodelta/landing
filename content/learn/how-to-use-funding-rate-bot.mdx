### Exchange Account Setup

Each exchange has slightly different API setup procedures:

**Binance:**
1. Account → API Management → Create API
2. Enable "Enable Spot & Margin Trading"
3. Restrict IP (optional but recommended)
4. Never enable "Enable Withdrawals"

**Bybit:**
1. Account & Security → API → Create API Key
2. Select "Trade" permissions
3. Disable "Withdrawal" and "Transfer"
4. Save API key and secret

**Gate.io, KuCoin, BingX:** Similar process - enable trading, disable withdrawals.

⚠️ **Security reminder:** Only grant trading permissions. Never enable withdrawal permissions for any trading bot.

## Common Issues and Solutions

### "Insufficient Balance" Error
**Cause:** Not enough USDT in spot or futures wallet
**Solution:** Transfer funds between wallets or deposit more capital
**Prevention:** Keep balance split 50/50 between spot and futures wallets

### "API Permissions" Error
**Cause:** API key lacks required trading permissions
**Solution:** Recreate API key with spot and futures trading enabled
**Check:** Verify IP whitelist if you enabled IP restrictions

### "Rate Changed" Warning
**Cause:** Funding rate moved significantly between alert and execution
**Bot behavior:** Warns you and asks for confirmation before proceeding
**Decision:** Proceed if still profitable, skip if rate compressed too much

### "Partial Fill" Notification
**Cause:** Large position split across multiple orders due to liquidity
**Bot behavior:** Continues attempting to fill remaining size
**Action:** Monitor progress, consider reducing position size for future trades

### "Settlement Missed" Alert
**Cause:** Position opened too close to settlement time
**Risk:** Holding through next funding period with unknown rate
**Strategy:** Only open positions with >30 minutes until settlement

### How much money do I need to start?
**Minimum:** $25 (exchange position limits)
**Recommended:** $100-500 for meaningful profits

### Is this really risk-free?
**No.** "Delta-neutral" means no directional price risk, but other risks exist:
- Exchange counterparty risk (exchange insolvency)
- Execution risk (timing delays between spot and perp orders)  
- Funding rate reversal (rates can flip negative)
- Technical risk (bot downtime, API issues)

### How often do extreme rates appear?
**Depends on volatility.** During calm markets: 0-2 opportunities per week. During volatile periods: multiple opportunities daily.
**Historical average:** 3-5 profitable opportunities per week across all monitored exchanges.

### Can I use multiple exchanges simultaneously?
**Yes.** Connect all supported exchanges for maximum coverage. More exchanges = more opportunities to capture cross-exchange rate spreads.

### What happens if funding rate goes negative?
**Auto-exit:** Bot monitors rate changes and warns if funding flips negative before settlement
**User choice:** Continue holding (now paying funding instead of receiving) or close position
**Risk management:** Built-in stops prevent holding through severely negative funding

### Do I pay taxes on funding rate profits?
**Consult tax professional.** Generally treated as trading income in most jurisdictions. Keep records of all settlements for tax reporting.
