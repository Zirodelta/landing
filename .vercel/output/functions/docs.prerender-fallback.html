Engine Architecture

Zirodelta runs an autonomous funding rate arbitrage engine called FarmingLoop. It continuously scans exchanges for profitable funding rate spreads and manages positions algorithmically.

Pipeline Overview

0

Collect

Connects to dozens of exchanges in parallel, fetching current funding rates, payment intervals, and next funding timestamps for every perpetual futures pair. Intervals are fetched dynamically per pair — never hardcoded. If an exchange changes a pair from 8h to 4h settlements, the engine picks it up automatically.

1

Scan & Score

Fetches market depth data — open interest, 24h volume, bid/ask spread — and calculates maximum safe position sizes per pair per exchange.

Then computes cross-exchange delta opportunities with settlement-aware scoring:

Matched Intervals

Both exchanges pay on the same schedule. Standard spread calculation — the exchange with the higher rate becomes the short leg (collecting side).

Mismatched Intervals

Conservative spread estimation. The fastest-settling exchange is assigned to the collecting side — so you bank real money sooner and re-evaluate each epoch:

  • Both rates positive → shorts collect → fastest = short
  • Both rates negative → longs collect → fastest = long
  • Mixed signs → both sides collect → best case scenario

Each opportunity includes break-even analysis: how many funding sessions needed to recover entry fees.

2

Think

Runs 6 sequential safety checks before committing capital:

  1. Consecutive failure detection — back off after repeated failures
  2. Existing position health — rebalance if spreads compressed
  3. Opportunity availability — hold if no profitable deltas
  4. Quality threshold — minimum annualized spread gate
  5. Position count — concurrent hedge limits
  6. Capital utilization — deployment ceiling

Decision output: enter, rebalance, or hold.

3

Rebalance

If existing positions have deteriorated:

  • Funding flip (spread went negative) → urgency routing to emergency close
  • Spread compressed (below threshold) → smart exit via TWAP

Uses the same interval-aware spread formula as Phase 1 — no math contradictions between entry and exit evaluation.

Minimum Hold Period

No position is closed until at least one collecting-side settlement has occurred. The collecting side is always the fastest-settling exchange — so if the short leg settles hourly and the long leg every 8 hours, the engine waits at least 1 hour before evaluating.

After collecting, if the spread looks bad: close before the slow side pays — don't wait to lose money. Emergency overrides (hard negative spreads) bypass the hold period entirely.

4

Allocate

Position-aware greedy knapsack allocation. Before building a plan, the engine fetches all open hedges and applies three layers of intelligence:

Route Deduplication

Won't open the same (symbol, short exchange, long exchange) twice. Different venue pairs for the same asset are allowed — they're independent routes with different risk profiles.

Capacity Subtraction

Deducts already-deployed capital from available exchange capacity before allocating new positions. Prevents over-concentration on a single exchange or symbol.

Rotation Scoring

When a better opportunity appears for the same asset on different venues, the engine calculates full rotation cost: exit fees + entry fees + withdrawal + deposit + bridge fees. Only rotates if the spread improvement recovers all costs within a defined payback period.

The allocation is additive only — it fills empty slots and proposes rotations. It never closes profitable positions. Closing is exclusively handled by Rebalance and the Real-Time Guard.

5

Track

Records actual funding payments received on all open positions. Not projected revenue — real settled cash.

6

Split

Revenue distribution depends on the engine class:

Class A — Proprietary

Our own capital, our own engine. No split — 100% retained. This is the core Beta-X1 instance.

Class B — Vault

Pooled external capital. Net revenue after costs is distributed:

60%

Depositor Yield

7

Record

Logs the complete cycle outcome — action taken, deltas found, positions opened/closed, capital utilization, funding collected, errors, duration. Feeds the failure detection in Phase 2 of the next cycle.

Real-Time Guard

Independent of the 30-minute loop, FundingGuardService monitors all open positions via exchange websockets in real-time. If a rate flips between cycles, it triggers an immediate exit — no waiting for the next scan.

Key Design Principles

Cash Flow Velocity > APR

The engine prioritizes real money movement speed over annualized rate projections. A lower rate with hourly settlement beats a higher rate with 8-hour settlement because you collect cash sooner and re-evaluate faster.

Interval-Aware Spread Math

When pairing exchanges with different payment schedules, rates are normalized to a common timeframe before comparison. The engine computes the actual net earning from each side of the hedge — what the collecting side earns minus what the paying side costs.

No Entry/Exit Contradictions

All spread calculations — scoring, allocation, rebalance, guard — use the identical interval-aware formula. The engine never enters a position using one model and evaluates it with another.

Additive Allocation, Subtractive Exit

The farming loop only adds new positions or rotates when the math proves it. Closing positions is exclusively handled by rebalance and the real-time guard. This separation prevents churn — the engine never closes a profitable position to reopen the same thing.

Rotation Must Pay for Itself

Switching from one exchange route to another costs real money: trading fees, withdrawal, deposit, and bridge costs. The engine only rotates if the spread improvement recovers the full rotation cost within a strict payback window. Small improvements are not worth the friction.

Zero Hardcoded Assumptions

Funding intervals, settlement schedules, and exchange parameters are all fetched dynamically. The engine never assumes a pair pays every 8 hours — it asks the exchange every cycle. Data quality at the source is non-negotiable.
